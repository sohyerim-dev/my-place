# My Place 기획서

## 1. 서비스 개요

**서비스명:** My Place (마이 플레이스)

**한 줄 소개:**
사진을 통해 장소를 기록하고 발견하는 개인 맞춤 위치 기반 커뮤니티 서비스

My Place는 사용자가 직접 촬영한 사진을 중심으로 장소를 공유하고,
지도 기반으로 저장·탐색할 수 있는 사진 중심 플랫폼이다.

텍스트 리뷰보다 직관적인 사진을 통해
사용자는 장소의 분위기를 파악하고
자신만의 장소 기록을 구축할 수 있다.

## 2. 기획 배경

기존 장소 탐색 서비스는 다음과 같은 한계를 가진다.

- 지도 서비스는 정보 중심으로 감성적 탐색이 어렵다
- SNS는 사진 중심이지만 장소 정보 활용성이 낮다
- 리뷰 플랫폼은 텍스트 중심으로 탐색 피로도가 높다

사용자는 다음과 같은 니즈를 갖고 있다.

- 사진으로 장소 분위기를 먼저 확인하고 싶다
- 마음에 드는 장소를 저장하고 관리하고 싶다
- 주변에서 갈 만한 곳을 쉽게 찾고 싶다
- 나만의 장소 기록을 만들고 싶다

My Place는
사진 기반 탐색 + 위치 정보 + 개인 아카이브 기능을 결합하여
이 문제를 해결한다.

## 3. 서비스 목표

- 감성 중심 장소 탐색 경험 제공
- 사용자 참여 기반 장소 데이터 구축
- 개인 맞춤 장소 기록 플랫폼 형성
- 온라인 경험을 실제 방문으로 연결

## 4. 주요 기능

### 4.1 메인 피드

사용자가 등록한 게시글을 피드 형태로 노출한다.
별도의 게시글 상세 페이지 없이 피드에서 바로 모든 정보를 확인할 수 있다.

게시글 카드 구성:

- 사진 슬라이드
- 장소명 (클릭 시 장소 상세 페이지로 이동)
- 본문 (설명/메모)
- 태그
- 좋아요 / 저장 버튼
- 댓글 — `comment(n)` 형태로 개수만 표시, 클릭 시 아코디언으로 펼침

정렬:

- 최신순 / 인기순

### 4.2 사진 기반 게시글 작성

사용자는 장소와 연결된 사진을 업로드하여 게시글을 작성할 수 있다.

게시글 구성 요소:

- 사진 (필수)
- 장소 정보 (지도 기반 선택)
- 간단한 설명 또는 메모
- 네이버 플레이스 URL (선택)
- 태그

### 4.3 댓글

게시글에 댓글을 작성할 수 있다.

- 피드에서 `comment(n)` 클릭 시 아코디언 방식으로 펼침
- 접기/펼치기로 피드 흐름 유지

### 4.4 좋아요 및 저장 기능

- 좋아요: 콘텐츠에 대한 감정 반응
- 저장: 나중에 방문하거나 기록하기 위한 북마크
- 저장된 장소는 개인 공간에서 관리된다.

### 4.5 장소 상세 페이지

피드에서 장소명을 클릭하면 해당 장소의 상세 정보를 확인할 수 있다.

포함 요소:

- 지도 (네이버 지도)
- 장소명
- 주소
- 네이버 플레이스 링크 (선택)

### 4.7 My Place — 개인 장소 지도

사용자가 저장하거나 등록한 장소를 지도 위에서 확인할 수 있는 기능

포함 요소:

- 내가 등록한 장소
- 저장한 장소
- 카테고리별 필터
- 방문 기록 관리

개인의 취향과 생활 동선이 반영된 '나만의 장소 지도'를 형성한다.

### 4.8 GPS 기반 주변 탐색

현재 위치를 기준으로 주변 사용자들이 등록한 장소를 탐색할 수 있다.

탐색 방식:

- 지도 위 표시
- 거리순 정렬
- 최신순 / 인기순 정렬
- 사진 썸네일 기반 표시

사용자는 목적 없이도 주변에서 갈 만한 장소를 발견할 수 있다.

### 4.9 사진 기반 지도 표시

지도 위에 단순 핀 대신 장소 대표 사진을 썸네일 형태로 표시한다.

이를 통해 사용자는 텍스트 없이도 장소 분위기를 직관적으로 파악할 수 있다.

### 4.10 이번주 핫플

최근 사용자 반응이 높은 장소를 별도 페이지(/hot)에서 확인할 수 있다.

선정 기준 예시:

- 최근 일정 기간 저장 수
- 조회 수
- 게시글 증가량

이를 통해 현재 인기 있는 장소를 빠르게 확인할 수 있다.

## 5. 화면 구조

### 상단 헤더 (홈 전용)

```
[+]    [로고]    [프로필]
```

- 좌측 `+` : 게시글 작성
- 중앙 : 로고 (카메라 아이콘)
- 우측 : 프로필 페이지 이동

### 하단 네비게이션 (전체 공통)

```
홈 (피드)  |  핫플  |  탐색  |  마이 플레이스
```

## 6. 사용자 흐름

### 장소 탐색 흐름

메인 피드 → 게시글 확인 → 장소명 클릭 → 장소 상세 → 방문

또는

GPS 탐색 → 주변 사진 확인 → 장소 선택 → 정보 확인 → 방문

또는

핫플 → 인기 장소 확인 → 게시글 확인 → 저장 → 방문

### 장소 기록 흐름

사진 촬영 → 게시글 업로드 → 장소 등록 → 개인 지도에 축적

## 7. 차별성

### 장소 중심 커뮤니티

- 기존 SNS: 사람 중심
- 지도 서비스: 정보 중심
- My Place: 장소 중심

### 사진 기반 탐색

텍스트 리뷰보다 직관적인 방식으로 장소를 선택할 수 있다.

### 개인 아카이브 기능

단순 공유를 넘어 개인의 장소 기록이 지속적으로 축적된다.

## 8. 기대 효과

- 새로운 장소 발견 경험 제공
- 지역 기반 커뮤니티 형성
- 사용자 참여형 데이터 축적
- 오프라인 활동 증가 유도
